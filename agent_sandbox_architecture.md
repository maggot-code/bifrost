# Agent 运维系统架构设计说明（Sandbox & Execution Model）

------------------------------------------------------------------------

# 1. 当前系统已实现能力

当前系统已经具备一个较为完整的"AI + 运维执行闭环"，核心能力如下：

## 1.1 人机协作运维（Interactive Mode）

系统支持用户与 Agent 在 Web 终端中协同操作远程设备：

-   用户通过 Web Terminal 登录远程设备（PTY）
-   FastAPI 负责建立 PTY 连接并持续读取输出
-   输出通过 WebSocket 推送至前端，实现实时终端体验
-   Agent 通过 MCP 调用 FastAPI 的命令注入接口执行命令
-   所有 Agent 执行前必须经过用户确认（Human-in-the-loop）

### 执行链路

User → MCP → User Approval → FastAPI → PTY → Remote Device

------------------------------------------------------------------------

## 1.2 自动运维（Agent-driven Mode）

系统支持基于告警触发的自动运维能力：

-   告警平台接收监控系统数据（Zabbix等）
-   告警推送至 Agent
-   Agent 自动分析告警
-   Agent 调用远程工具收集真实数据（CPU / 内存 / 磁盘 / 进程等）
-   Agent 生成分析报告与修复建议
-   结果回传用户

Alert → Agent → Remote Tool API → Analysis → Report → User

------------------------------------------------------------------------

## 1.3 已具备能力总结

-   PTY 实时远程终端执行
-   用户确认机制
-   WebSocket 实时回显
-   远程只读观测工具（Tool API）
-   MCP 工具调度层
-   告警驱动自动分析链路

------------------------------------------------------------------------

# 2. 引入 OpenSandbox 的方案（Agent Sandbox as a Tool）

引入 OpenSandbox 作为"Agent执行安全层"，不替代 PTY，而是补充能力。

## 2.1 引入目标

-   Agent 行为模拟
-   命令风险评估
-   不可信执行环境隔离
-   代码/脚本运行环境

------------------------------------------------------------------------

## 2.2 执行模型扩展

  类型              作用       是否影响真实系统
  ----------------- ---------- ------------------
  Remote Tool API   只读观测   否
  PTY               生产执行   是
  OpenSandbox       行为模拟   否

------------------------------------------------------------------------

## 2.3 扩展执行流

人机协作：

Agent → Tool/API → Sandbox（可选） → 用户确认 → PTY

自动运维：

Alert → Agent → Tool/API → Sandbox → 决策 → 执行/报告

------------------------------------------------------------------------

## 2.4 Sandbox 定位

Agent 的安全推演执行环境（Simulation Layer）

------------------------------------------------------------------------

# 3. OpenSandbox 关键定位

-   不是生产执行环境
-   不替代 PTY
-   不提供真实系统状态

## 核心职责

### 行为模拟

-   命令行为分析
-   操作类型识别

### 风险评估

-   是否危险操作
-   是否不可逆

### Agent 推理空间

-   执行预演
-   策略验证

------------------------------------------------------------------------

## 本质定义

行为后果模拟器，而非真实环境复制器

------------------------------------------------------------------------

# 4. 核心问题与解答

## 问题1：没有真实数据是否影响风险判断？

不影响。Sandbox 判断的是行为风险，不是数据结果。

## 问题2：Sandbox vs PTY区别？

-   PTY：真实执行
-   Sandbox：行为模拟

## 问题3：Sandbox作用？

-   预执行验证
-   风险分析
-   Agent推理辅助

## 问题4：是否替代Web Terminal？

不会。Web Terminal = 真实执行入口

------------------------------------------------------------------------

# 5. 最终架构

Agent Layer ↓ Execution Router ↓ Sandbox / Tool API / PTY

------------------------------------------------------------------------

## 三层体系

-   Tool API：观测
-   Sandbox：模拟
-   PTY：执行

------------------------------------------------------------------------

# 6. 设计原则

-   执行分离
-   风险前置
-   PTY唯一真实执行入口
